리다이렉트는 클라이언트측 처리!

--
아키텍처 -> 순서 -> 코드 -> 최종 작동(영상)

--

"리다이렉션"은 주로 백엔드에서 클라이언트에게 전달되는 명령 중 하나입니다. 프론트엔드와 백엔드 간에 협력하여 사용되는 경우가 많습니다.

주요 사용 사례는 다음과 같습니다:

1. **사용자 로그인 후 리다이렉션**: 사용자가 로그인 성공 후, 백엔드는 클라이언트에게 다음으로 이동할 URL을 알려주어, 클라이언트는 해당 URL로 이동합니다.

2. **폼 제출 후 리다이렉션**: 사용자가 양식을 제출하고, 백엔드에서 처리한 후에 특정 페이지로 리다이렉션을 수행할 수 있습니다.

3. **리소스 업로드 후 리다이렉션**: 파일 등을 업로드하고, 업로드가 성공하면 특정 페이지로 이동하도록 리다이렉션을 수행합니다.

리다이렉션은 서버에서 클라이언트에게 특정한 동작을 수행하도록 지시하는 방법 중 하나로, 서버가 클라이언트에게 특정 URL로 이동하라고 알리는 역할을 합니다. 클라이언트는 이 정보를 받아 이동을 수행하게 됩니다.
