
1. **HTML 기본 지식**: HTML(HyperText Markup Language)은 웹 페이지의 구조를 만드는 데 사용되는 마크업 언어입니다. 웹 페이지의 제목, 문단, 이미지, 링크, 리스트 등은 모두 HTML 태그를 사용하여 표현됩니다. 예를 들어, `<h1>`은 가장 큰 제목을, `<p>`는 문단을, `<img>`는 이미지를, `<a>`는 링크를 나타냅니다.

2. **JavaScript 기본 지식**: JavaScript는 웹 페이지에 동적인 기능을 추가하는 데 사용되는 프로그래밍 언어입니다. JavaScript를 사용하면 사용자의 액션에 반응하거나, 웹 페이지의 내용을 동적으로 변경하거나, 서버로부터 데이터를 가져와서 웹 페이지에 표시하는 등의 작업을 할 수 있습니다. JavaScript는 변수, 함수, 객체, 배열, 조건문, 반복문, 이벤트, 비동기 처리 등 다양한 개념들을 포함하고 있습니다.

3. **jQuery 라이브러리 이해**: jQuery는 JavaScript 라이브러리 중 하나로, JavaScript의 복잡한 작업들을 간단하게 처리할 수 있게 도와줍니다. 예를 들어, DOM(Document Object Model) 조작, 이벤트 핸들링, 애니메이션 처리, Ajax 처리 등을 jQuery를 사용하여 쉽게 할 수 있습니다.

4. **HTTP 프로토콜 이해**: HTTP(HyperText Transfer Protocol)는 웹에서 데이터를 주고받는 데 사용되는 프로토콜입니다. 클라이언트(보통 웹 브라우저)는 HTTP 요청을 서버에 보내고, 서버는 HTTP 응답을 클라이언트에 보냅니다. HTTP 요청은 메서드(GET, POST 등), URL, 헤더, 바디 등으로 구성되고, HTTP 응답은 상태 코드(200, 404, 500 등), 헤더, 바디 등으로 구성됩니다.

5. **RESTful API 이해**: REST(Representational State Transfer)는 웹 서비스를 설계하는 방법론 중 하나입니다. RESTful API는 HTTP 프로토콜의 메서드와 URL을 이용하여 리소스를 명시하고, 해당 리소스에 수행할 작업을 정의합니다. 일반적으로 GET은 리소스를 조회, POST는 리소스를 생성, PUT은 리소스를 수정, DELETE는 리소스를 삭제하는 데 사용됩니다.

6. **OAuth 인증 흐름 이해**: OAuth(Open Authorization)는 사용자의 인증 정보를 다른 애플리케이션과 안전하게 공유하는 방식입니다. OAuth를 이용한 소셜 로그인(카카오 로그인 등)은 사용자 인증 후에 인증 코드를 받아와서 액세스 토큰을 얻는 과정을 거칩니다. 액세스 토큰은 사용자의 인증 상태를 나타내는 토큰으로, 이 토큰을 이용하여 사용자의 정보를 가져오거나 사용자를 대신하여 작업을 수행할 수 있습니다.
